## 1. App Overview

### What It Does

GreenLedger is an AI-powered climate disclosure report generator for Australian businesses. Users enter their company details and greenhouse gas emissions data, and the app generates a complete **AASB S2** (aligned with **IFRS S2**) compliance report covering all four mandatory pillars:

1. **Governance** — Board oversight of climate risks
2. **Strategy** — Climate risks/opportunities over short, medium, and long term
3. **Risk Management** — Processes for identifying and managing climate risks
4. **Metrics & Targets** — GHG emissions data (Scope 1, 2, 3) and reduction targets

### Who It's For

- Australian businesses required to comply with AASB S2 climate disclosure standards
- CFOs, sustainability officers, and compliance teams
- SMEs through to ASX-listed firms
- Regional manufacturers, mining companies, logistics firms, and professional services

### Core Value Proposition

> "Climate disclosure reports in minutes, not months."

Instead of spending months and tens of thousands on sustainability consultants, businesses enter their emissions data and get a board-ready AASB S2 report generated by AI in 30-60 seconds.

### Key Features

- **NGA Emission Factors:** Built-in Australian National Greenhouse Accounts factors for automatic CO2e calculations
- **All Scopes Covered:** Scope 1 (direct), Scope 2 (electricity), Scope 3 (value chain) — 23 source types
- **State-Specific Electricity:** Different emission factors for each Australian state/territory grid
- **AI-Powered Report Generation:** GPT-4o generates all four AASB S2 sections tailored to the company's actual data
- **Board-Ready HTML Reports:** Professional formatted reports that can be printed or saved as PDF
- **No Authentication Required:** Open access — no login, no signup. Just start entering data.

### `POST /api/companies/:companyId/emissions`

Add one or more emission records to a company. Accepts a single object or an array.

**Request Body (single):**
```json
{
  "category": "scope2",
  "source_type": "electricity_nsw",
  "quantity": 50000,
  "unit": "kWh",
  "description": "Office electricity - Sydney HQ",
  "period_start": "2024-07-01",
  "period_end": "2025-06-30",
  "data_source": "manual"
}
```

**Request Body (batch):**
```json
[
  { "category": "scope1", "source_type": "natural_gas", "quantity": 1200, "unit": "GJ" },
  { "category": "scope2", "source_type": "electricity_vic", "quantity": 35000, "unit": "kWh" }
]
```

**Required fields per item:** `category`, `source_type`, `quantity`, `unit`

**Server-side behavior:**
1. Validates the company exists
2. For each item, calls `calculateEmissions(category, source_type, quantity)`
3. If a matching NGA factor exists, auto-calculates `co2e_tonnes` and `emission_factor`
4. If no matching factor, falls back to `item.co2e_tonnes` or `item.emission_factor` from the request body, or `null`
5. Inserts into `emissions_data`

**Response (200):**
```json
{
  "emissions": [
    {
      "id": 1,
      "company_id": 1,
      "category": "scope2",
      "source_type": "electricity_nsw",
      "quantity": "50000.0000",
      "unit": "kWh",
      "emission_factor": "0.79000000",
      "co2e_tonnes": "39.5000",
      "period_start": "2024-07-01",
```

### `POST /api/companies/:companyId/reports/generate`

Generate a full AASB S2 report for a company. This is **synchronous** — the response returns when the report is fully generated (30-60 seconds).

**Flow:**
1. Fetch company record
2. Fetch all emissions for the company
3. Calculate scope totals (Scope 1, 2, 3, Total) — rounded to 4 decimal places
4. Insert a report record with `status = 'generating'`
5. Call `generateReport(company, emissionsData, totals)` which makes **4 parallel OpenAI API calls** (one per AASB S2 section)
6. Build the full HTML report from the 4 sections
7. Update the report record with `status = 'completed'`, HTML content, section texts, and totals
8. Return the completed report

**Request Body:** None required

| `employee_commute_car` | 0.171 | km | Employee commute by car |
| `waste_landfill` | 1.1 | tonnes | Waste to landfill |
| `waste_recycled` | 0.021 | tonnes | Waste recycled |
| `water_supply` | 0.394 | kL | Water supply & treatment |
| `paper` | 0.919 | tonnes | Paper consumption |

### Calculation Formula

```
CO2e (kg) = quantity * emission_factor
CO2e (tonnes) = CO2e (kg) / 1000
```

**Rounding:** Results are rounded to 4 decimal places using: `Math.round(value * 10000) / 10000`

### Complete Calculation Code

```javascript
function calculateEmissions(category, sourceType, quantity) {
  const factors = EMISSION_FACTORS[category];
  if (!factors || !factors[sourceType]) return null;

  const ef = factors[sourceType];
  // Factor is in kg CO2-e per unit, convert to tonnes
  const co2eKg = quantity * ef.factor;
  const co2eTonnes = co2eKg / 1000;

  return {
    co2e_tonnes: Math.round(co2eTonnes * 10000) / 10000,
    emission_factor: ef.factor,
    unit: ef.unit,
    description: ef.description
  };
}
```

### Scope Totals Calculation (Server-Side)

When generating a report, the server sums emissions by scope:

```javascript
const totals = emissionsData.reduce((acc, e) => {
  const val = parseFloat(e.co2e_tonnes) || 0;
  if (e.category === 'scope1') acc.scope1 += val;
  else if (e.category === 'scope2') acc.scope2 += val;
  else if (e.category === 'scope3') acc.scope3 += val;
  acc.total += val;
  return acc;
}, { scope1: 0, scope2: 0, scope3: 0, total: 0 });

// Round to 4 decimal places
totals.scope1 = Math.round(totals.scope1 * 10000) / 10000;
totals.scope2 = Math.round(totals.scope2 * 10000) / 10000;
totals.scope3 = Math.round(totals.scope3 * 10000) / 10000;
totals.total = Math.round(totals.total * 10000) / 10000;
```

### `getSourceTypes()` Function

Returns all source types in a frontend-friendly format:

```javascript
function getSourceTypes() {
  const types = {};
  for (const [category, sources] of Object.entries(EMISSION_FACTORS)) {
    types[category] = Object.entries(sources).map(([key, val]) => ({
      key,
      unit: val.unit,
      description: val.description,
      factor: val.factor
    }));
  }
  return types;
}
```

## 16. Migration System

### Overview

Custom migration runner in `migrate.js`. Not using any migration library — pure Node.js + `pg`.

### How It Works

1. Creates `_migrations` tracking table (idempotent)
2. Reads all `.js` files from `migrations/` folder, sorted alphabetically (timestamp prefix ensures order)
3. Checks which migrations have already been applied (by name in `_migrations`)
4. Runs new migrations in a transaction (BEGIN → up() → INSERT into _migrations → COMMIT)
5. On failure: ROLLBACK and throw error

### Migration File Format

```javascript
module.exports = {
  name: 'create_greenledger_tables',  // Must be unique
  up: async (client) => {
    // client is a pg Client (from pool.connect())
    await client.query(`CREATE TABLE IF NOT EXISTS ...`);
  }
};
```

### File Naming Convention

```
{unix_timestamp_ms}_{descriptive_name}.js
```

Example: `1709524800000_create_greenledger_tables.js`

### Current Migrations

| File | Name | Description |
|------|------|-------------|
| `1709524800000_create_greenledger_tables.js` | `create_greenledger_tables` | Creates `companies`, `emissions_data`, `reports` tables with indexes |

All `CREATE TABLE` statements use `IF NOT EXISTS` for idempotency. All `CREATE INDEX` statements use `IF NOT EXISTS`.

### Migration Runner Code

```javascript
const { Pool } = require('pg');
const fs = require('fs');
const path = require('path');

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  ssl: process.env.DATABASE_URL.includes('localhost') ? false : { rejectUnauthorized: false },
});

async function migrate() {
  const client = await pool.connect();
  try {
    // Create tracking table
    await client.query(`
      CREATE TABLE IF NOT EXISTS _migrations (
        id SERIAL PRIMARY KEY,
        name VARCHAR(255) NOT NULL UNIQUE,
        applied_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
      )
    `);

    // Run folder migrations
    const migrationsDir = path.join(__dirname, 'migrations');
    if (!fs.existsSync(migrationsDir)) return;

    const files = fs.readdirSync(migrationsDir).filter(f => f.endsWith('.js')).sort();
    const applied = await client.query('SELECT name FROM _migrations');
    const appliedNames = new Set(applied.rows.map(r => r.name));

    for (const file of files) {
      const migration = require(path.join(migrationsDir, file));
      const name = migration.name || file.replace('.js', '');
      if (appliedNames.has(name)) continue;

      await client.query('BEGIN');
      await migration.up(client);
      await client.query('INSERT INTO _migrations (name) VALUES ($1)', [name]);
      await client.query('COMMIT');
    }
  } finally {
    client.release();
    await pool.end();
  }
}

migrate().catch(err => { console.error(err.message); process.exit(1); });
```
