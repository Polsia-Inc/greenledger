13. Step-by-Step User Journey

1. Landing Page (/)

User arrives at the dark-themed landing page. They see:

  • Hero section explaining what GreenLedger does
  • Feature cards showing capabilities
  • AASB S2 framework explanation with the 4 pillars
  • Multiple CTA buttons all pointing to /app

2. Click "Get Started"

User navigates to /app. The dashboard loads with Step 1 (Company Profile) active.

3. Create Company Profile (Step 1)

  • User fills in company details (Company Name required, Financial Year required)
  • Clicks "Save & Continue"
  • Spinner shows "Saving..."
  • Company is saved via POST /api/companies
  • Toast: "Company profile saved"
  • Auto-navigates to Step 2 after 500ms

OR if returning user:

  • Existing companies appear in "Your Companies" card
  • User clicks "Select" on an existing company
  • Form populates with company data

4. Enter Emissions Data (Step 2)

  • User sees 4 metric cards (all showing 0 initially)
  • User fills in the "Add Emission Source" form:
    • Selects Scope (Scope 1, 2, or 3)
    • Source Type dropdown populates based on scope
    • Unit field auto-fills (readonly)
    • Enters Quantity
    • Optionally sets Period Start/End and Description
  • Clicks "Add Emission"
  • Emission saved via POST /api/companies/:id/emissions
  • Table updates showing the new record with calculated tCO2e
  • Metric cards update with new totals
  • User repeats for all emission sources
  • User clicks "Continue to Report →"

5. Generate Report (Step 3)

  • User clicks "Generate AASB S2 Report"
  • Button disables, spinner shows "Generating report (30-60s)..."
  • POST /api/companies/:id/reports/generate fires
  • Server creates report record with status: 'generating'
  • Server makes 4 parallel OpenAI API calls
  • After ~30-60 seconds, server responds with completed report
  • Client starts polling every 3 seconds
  • When status: 'completed':
    • Toast: "Your AASB S2 report is ready!"
    • Report appears in list with green "completed" badge
    • "View Report" button appears

6. View/Print Report

  • User clicks "View Report"
  • Opens /api/reports/:id/html in a new browser tab
  • Full AASB S2 report renders with:
    • Dark header with company name, ABN, financial year
    • 4 summary cards with scope totals
    • Governance section
    • Strategy section
    • Risk Management section
    • Metrics & Targets section (includes data tables)
    • Footer with compliance statement
  • User can Ctrl+P / Cmd+P to print or save as PDF (print styles are included)

14. Error Handling & Validation

Server-Side Validation

Endpoint Validation Error Response
POST /api/companies name and financial_year required 400: "Company name and financial year are required"
POST /api/companies/:id/emissions Company must exist 404: "Company not found"
POST /api/companies/:id/emissions Each item needs category, source_type, quantity, unit 400: "category, source_type, quantity, and unit are required for each item"
POST /api/companies/:id/reports/generate Emissions must exist 400: "No emissions data found. Add emissions data before generating a report."
GET /api/reports/:id/html Status must be completed 400: "Report not ready yet"

Server-Side Error Pattern

All route handlers use try/catch with generic 500 errors:

try {
  // ... handler logic
} catch (err) {
  console.error('Descriptive error:', err);
  res.status(500).json({ error: 'Failed to {action}' });
}

Report Generation Failure

If AI generation fails:

  1. Report status updated to 'failed' in database
  2. Response includes { report: { id, status: 'failed' }, error: genErr.message }
  3. Client shows "Generation failed. Try again." status text

Client-Side Error Handling

  • All fetch() calls are wrapped in try/catch
  • Errors logged to console.error
  • User-facing errors shown via toast() notification
  • Form buttons re-enabled after error

Client-Side Validation

  • HTML5 required attribute on Company Name, Financial Year, Scope, Source Type, Quantity
  • type="email" on Contact Email field
  • type="number" with min="0" and step="any" on Quantity field
  • maxlength="20" on ABN field
  • No company check: Steps 2 and 3 show "Create a company first" empty state if currentCompany is null

16. Migration System

Overview

Custom migration runner in migrate.js. Not using any migration library — pure Node.js + pg.

How It Works

  1. Creates _migrations tracking table (idempotent)
  2. Reads all .js files from migrations/ folder, sorted alphabetically
  3. Checks which migrations have already been applied (by name in _migrations)
  4. Runs new migrations in a transaction (BEGIN → up() → INSERT into _migrations → COMMIT)
  5. On failure: ROLLBACK and throw error

Migration File Format

module.exports = {
  name: 'create_greenledger_tables',  // Must be unique
  up: async (client) => {
    // client is a pg Client (from pool.connect())
    await client.query(`CREATE TABLE IF NOT EXISTS ...`);
  }
};

File Naming Convention

{unix_timestamp_ms}_{descriptive_name}.js

Example: 1709524800000_create_greenledger_tables.js

Current Migrations

File Name Description
1709524800000_create_greenledger_tables.js create_greenledger_tables Creates companies, emissions_data, reports tables with indexes

All CREATE TABLE statements use IF NOT EXISTS for idempotency. All CREATE INDEX statements use IF NOT EXISTS.

Appendix B: Full Report HTML Template

The buildReportHtml() function generates a complete self-contained HTML document. The inline CSS is included in the report output.

Key implementation detail: The function uses JavaScript template literals to inject:

  • company.name — in title, header, and ABN display
  • company.abn — conditionally shown with " | ABN: " prefix
  • company.financial_year — in header meta
  • totals.scope1/scope2/scope3/total — formatted with .toFixed(1) in summary cards
  • sections.governance/strategy/riskManagement/metricsTargets — raw HTML from OpenAI injected directly
  • Generated date — formatted as "15 January 2026" (en-AU locale)
